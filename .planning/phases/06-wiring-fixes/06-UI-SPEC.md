## Interaction Contracts

### Toast Notification Lifecycle

```
SSE event received
  -> Create toast DOM element
  -> Append to document.body
  -> requestAnimationFrame -> add 'visible' class (trigger entry animation)
  -> setTimeout 6000ms -> remove 'visible' class (trigger exit animation)
  -> setTimeout 300ms -> remove DOM element
```

**Stacking behavior:** If a second toast appears while the first is still visible, the new toast appears below the existing one. Each toast manages its own dismiss timer independently. Use `bottom` offset calculation based on number of existing `.app-toast` elements.

### Share Toggle on Reload

```
Page load / navigation to /travel/{id}
  -> apiGetTravel(id) fetches plan from backend
  -> Backend now returns share_token in response
  -> showTravelGuide(plan) reads plan.share_token
  -> _renderShareToggle(travelId, shareToken) renders toggle in correct state
```

No visual changes — the toggle already works correctly when `share_token` is provided. This fix is purely data-wiring.

### Hints Input in Replace-Stop Flow

```
User opens replace-stop dialog
  -> Hints input visible above tabs (shared section)
  -> User optionally types hints (e.g., "mehr Strand")
  -> User clicks search tab "Neue Suche" button
  -> _doSearchReplace() reads value from #replace-stop-hints
  -> Sends hints in API request body
  -> StopOptionsFinderAgent receives hints as extra_instructions
```

### Tag Pills Render Flow

```
Agent generates stop with tags: ["Strand", "Kultur", "Wandern"]
  -> Tags stored in StopOption.tags
  -> Tags carried through orchestrator to TravelStop.tags
  -> Frontend reads stop.tags array
  -> Renders .stop-tag-pill elements (existing code, guide.js:1084-1087)
```
