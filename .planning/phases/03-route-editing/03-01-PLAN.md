**Create `backend/utils/route_edit_helpers.py`** with these functions extracted/adapted from replace_stop_job.py:

1. `async def recalc_segment_directions(stops: list, index: int, start_location: str) -> None` — Recalculates Google Directions for the segment arriving at `stops[index]`. Uses `google_directions_with_ferry()` (not `google_directions_simple`) per D-10 and Phase 2 pitfall 6 (ferry-aware recalc). If `index > 0`, origin is `f"{stops[index-1]['region']}, {stops[index-1].get('country', '')}"`, else origin is `start_location`. Sets `stops[index]["drive_hours_from_prev"]` = `round(hours, 1)` and `stops[index]["drive_km_from_prev"]` = `round(km)` when `hours > 0`.

2. `async def recalc_all_segments(stops: list, start_location: str) -> None` — Calls `recalc_segment_directions` for every stop index 0..len-1. Used by reorder (recalc ALL per research recommendation). First stop (index 0) uses `start_location` as origin.

3. `async def recalc_arrival_days(stops: list, from_index: int = 0) -> None` — Sets `stops[0]["arrival_day"] = 1` if `from_index == 0`. Then for each stop from `max(1, from_index)` onward: `stops[i]["arrival_day"] = stops[i-1]["arrival_day"] + stops[i-1].get("nights", 1) + 1` (the +1 is drive day between stops).

4. `async def run_day_planner_refresh(plan: dict, stops: list, request, job_id: str) -> None` — Re-runs `DayPlannerAgent(request, job_id).run()` with all stops, accommodations, and research data. Updates `plan["day_plans"]`, `plan["cost_estimate"]`, `plan["google_maps_overview_url"]` in-place. Wraps in try/except — on failure, logs WARNING but does not raise (matches replace_stop_job.py lines 231-235).

5. `async def run_research_pipeline(stop: dict, request, job_id: str, plan: dict) -> None` — Runs the full research pipeline for a single stop (Activities, Restaurants, Images in parallel via asyncio.gather, then TravelGuide, then Accommodation). Updates `stop` dict in-place with results. Follows replace_stop_job.py lines 140-201 exactly. Uses `get_setting("budget.accommodation_pct")` for budget calculation.

All functions use type hints. All logging via `debug_logger.log(LogLevel.X, msg, job_id=job_id, agent="RouteEdit")`.

```python
import json
import os

EDIT_LOCK_TTL = 300  # 5 minutes

def _get_redis():
    try:
        from main import redis_client
        return redis_client
    except Exception:
        import redis as redis_lib
        return redis_lib.from_url(os.getenv("REDIS_URL", "redis://localhost:6379"))

def acquire_edit_lock(travel_id: int) -> bool:
    r = _get_redis()
    if hasattr(r, 'set'):
        return bool(r.set(f"edit_lock:{travel_id}", "1", nx=True, ex=EDIT_LOCK_TTL))
    # InMemoryStore fallback: always allow (single process)
    return True

def release_edit_lock(travel_id: int) -> None:
    r = _get_redis()
    if hasattr(r, 'delete'):
        r.delete(f"edit_lock:{travel_id}")
```

**Create `backend/tasks/remove_stop_job.py`** (per D-03, D-04, D-05, CTL-01):

```python
# Structure follows replace_stop_job.py exactly
import asyncio, json, os, sys, traceback
sys.path.insert(0, os.path.dirname(os.path.dirname(os.path.abspath(__file__))))
from tasks import celery_app

def _get_store(): ... # same as replace_stop_job.py

async def _remove_stop_job(job_id: str):
    from utils.route_edit_helpers import recalc_segment_directions, recalc_arrival_days, run_day_planner_refresh
    from utils.route_edit_lock import release_edit_lock
    from utils.debug_logger import debug_logger, LogLevel
    from utils.travel_db import get_travel, update_plan_json
    from models.travel_request import TravelRequest

    store = _get_store()
    raw = store.get(f"job:{job_id}")
    if not raw: return
    job = json.loads(raw)
    travel_id = job["travel_id"]
    stop_index = job["stop_index"]
    user_id = job.get("user_id", 1)

    try:
        plan = await get_travel(travel_id, user_id)
        if not plan:
            await debug_logger.push_event(job_id, "job_error", None, {"error": f"Reise {travel_id} nicht gefunden"})
            return

        stops = plan.get("stops", [])
        req_data = plan.get("request", {})
        request = TravelRequest(**req_data)
        start_location = plan.get("start_location", "")

        await debug_logger.push_event(job_id, "remove_stop_progress", None, {
            "phase": "removing", "message": "Stopp wird entfernt..."
        })

        # Remove the stop
        removed = stops.pop(stop_index)

        # Reconnect: recalc directions for the stop now at stop_index (the successor)
        if stop_index < len(stops):
            await debug_logger.push_event(job_id, "remove_stop_progress", None, {
                "phase": "directions", "message": "Fahrzeiten werden neu berechnet..."
            })
            await recalc_segment_directions(stops, stop_index, start_location)

        # Rechain arrival days from removal point
        await recalc_arrival_days(stops, from_index=max(0, stop_index))

        # Re-run DayPlanner
        await debug_logger.push_event(job_id, "remove_stop_progress", None, {
            "phase": "day_planner", "message": "Tagesplaene werden aktualisiert..."
        })
        plan["stops"] = stops
        await run_day_planner_refresh(plan, stops, request, job_id)

        # Save
        await update_plan_json(travel_id, user_id, plan)

        await debug_logger.push_event(job_id, "remove_stop_complete", None, plan)
        job["status"] = "complete"
        job["result"] = plan
        store.setex(f"job:{job_id}", 86400, json.dumps(job))
    finally:
        release_edit_lock(travel_id)

@celery_app.task(name="tasks.remove_stop_job.remove_stop_job_task")
def remove_stop_job_task(job_id: str):
    asyncio.run(_remove_stop_job(job_id))
```

**Create `backend/tasks/add_stop_job.py`** (per D-06, D-07, D-08, CTL-02):

Similar structure. Job dict contains: `travel_id`, `insert_after_index` (position), `location_name`, `lat`, `lng`, `nights` (default 1), `user_id`.

Steps:
1. Load plan, validate insert position
2. Build new stop dict: `id = max(s["id"] for s in stops) + 1`, `region = location_name`, `country = "XX"`, `lat`, `lon = lng`, `nights`, clear research fields (top_activities=[], restaurants=[], travel_guide=None, accommodation=None, etc.)
3. Insert at `insert_after_index + 1` in stops list
4. Recalc directions for the new stop AND the next stop (if exists) via `recalc_segment_directions()`
5. Rechain arrival_days from insert point
6. Run full research pipeline via `run_research_pipeline(new_stop, request, job_id, plan)` — SSE events: `add_stop_progress` with phases "directions", "research", "guide", "accommodation", "day_planner"
7. Re-run DayPlanner via `run_day_planner_refresh()`
8. Save to DB, push `add_stop_complete` with full plan
9. Release edit lock in finally block

**Create `backend/tasks/reorder_stops_job.py`** (per D-09, D-10, CTL-03):

Job dict contains: `travel_id`, `old_index`, `new_index`, `user_id`.

Steps:
1. Load plan
2. Move stop: `moved = stops.pop(old_index)`, `stops.insert(new_index, moved)`
3. Reassign sequential IDs: `for i, s in enumerate(stops): s["id"] = i + 1`
4. Recalc ALL segments via `recalc_all_segments(stops, start_location)` (per research recommendation: recalc all, not just affected)
5. Rechain ALL arrival_days from index 0
6. Re-run DayPlanner
7. Save, push `reorder_stops_complete` with full plan
8. Release edit lock in finally block

SSE events: `reorder_stops_progress` with phases "reordering", "directions", "day_planner"
