## Spacing Scale

Declared values (multiples of 4 only, aligned to existing `--space-*` tokens in `styles.css`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline chip padding |
| sm | 8px | Breadcrumb segment gaps, compact element spacing |
| md | 16px | Day card internal padding, breadcrumb bar vertical padding |
| lg | 24px | Section padding, overview header padding |
| xl | 32px | Day card grid gap on desktop |
| 2xl | 48px | Between overview sections |
| 3xl | 64px | Page-level top/bottom breathing room |

Exceptions:
- Touch target for breadcrumb clickable segments: 48px height (closest standard token to Apple HIG 44px minimum — accessibility)
- Day card thumbnail height: 120px (fixed, not from scale — see Component Inventory)

Source: Existing `--space-*` vars in `styles.css` `:root`. Scale is rem-based (`--space-xs: 0.25rem` = 4px at 16px base). This spec uses px equivalents for precision.

## Component Inventory

New CSS classes required in `frontend/styles.css`. No new JS modules created — all additions extend existing guide modules.

### Unified Breadcrumb Bar

```
.guide-breadcrumb               — bar container, sticky below stats bar
.guide-breadcrumb__segment      — each clickable or static segment
.guide-breadcrumb__segment--active  — current (last) segment, non-clickable
.guide-breadcrumb__separator    — › separator between segments
```

Geometry:
- Height: 48px (touch target, accessibility — nearest multiple-of-4 above 44px)
- Background: `--bg-elevated` with 1px bottom border `--border-subtle`
- Horizontal padding: 16px (md)
- Vertical padding: 0 (vertically centered via flexbox)
- Hidden at overview level (display: none when no drill depth)
- Sticky: `position: sticky; top: [stats-bar-height]px; z-index: 10`

Source: CONTEXT.md D-06. Touch target: 48px (nearest multiple-of-4 to Apple HIG 44px minimum).

### Day Card Grid (Overview)

```
.day-cards-grid                 — grid container replacing existing day list
.day-card-v2                    — individual card (renamed to avoid collision with existing .day-card)
.day-card-v2__thumb             — thumbnail image area, 120px height
.day-card-v2__thumb-placeholder — fallback when no stop photo available
.day-card-v2__body              — text content area
.day-card-v2__meta              — stop count + drive time line
```

Geometry:
- Grid columns: 3-up on ≥1024px, 2-up on 480–1023px, 1-up below 480px
- Gap: 16px (md)
- Card border-radius: `--radius` (16px), consistent with existing cards
- Thumbnail: 120px height, `object-fit: cover`, `border-radius: --radius-sm` on top corners
- Thumbnail fallback: `--bg-secondary` background with centered region initial letter, 16px semibold, `--text-muted` color
- Card hover: existing `--lift-transition` / `--lift-y` / `--lift-shadow` pattern (already in codebase)
- Card cursor: pointer

Source: CONTEXT.md D-01. Columns/gap: Claude's discretion.

### Collapsible Details Section

```
.overview-collapsible           — container for trip analysis, budget, prose
.overview-collapsible__toggle   — clickable header row with chevron
.overview-collapsible__chevron  — ▾/▸ indicator (Unicode: ▾ collapsed → ▴ expanded)
.overview-collapsible__body     — collapsible content, height-based transition
```

Behavior:
- Default: collapsed (`max-height: 0; overflow: hidden`)
- Expanded: `max-height: 2000px` (safe upper bound for content)
- Transition: `max-height 0.3s cubic-bezier(0.4, 0, 0.2, 1)` (matches `--transition-default` easing)
- Chevron rotates 180° on expand: `transform: rotate(180deg); transition: transform 0.3s`

Source: CONTEXT.md D-02. Toggle design: Claude's discretion (accordion with chevron).

### Crossfade Transition Container

```
.guide-drill-panel              — wraps all drill-level content
.guide-drill-panel--exiting     — opacity 0, pointer-events none, applied during fade-out
.guide-drill-panel--entering    — opacity 0 → 1, applied on new content mount
```

Timing:
- Fade-out duration: 150ms, easing: `ease-in`
- Fade-in duration: 250ms, easing: `ease-out`
- Total perceived transition: ~250ms (overlap via sequential: out then in)
- Only `opacity` and `transform` animated (GPU-composited, no layout thrash)
- Scroll-to-top fires after fade-out completes, before fade-in begins (CONTEXT.md D-04)

Source: CONTEXT.md D-03. Exact timing and easing: Claude's discretion.

### Map Dimming (non-focused markers)

No new CSS class needed. Applied via Google Maps Marker `setOpacity(0.35)` call in `guide-map.js`. Focused markers remain at opacity 1.0. The day's polyline segment is highlighted with stroke weight 5px / color `--accent-primary` #7C6BE0; non-focused polyline segments render at opacity 0.25.

Source: CONTEXT.md D-09, D-10. Marker approach: Claude's discretion (opacity-only).

## Interaction Contract

### Drill-Down (click day card → day detail)

1. User clicks `.day-card-v2` — cursor pointer.
2. JS applies `.guide-drill-panel--exiting` to current panel (opacity → 0, 150ms ease-in).
3. After 150ms: scroll content panel to top (CONTEXT.md D-04).
4. New content rendered off-screen (opacity 0).
5. JS applies `.guide-drill-panel--entering` (opacity 0 → 1, 250ms ease-out).
6. Breadcrumb bar appears with `Übersicht › Tag N: [Titel]`.
7. URL pushed: `/travel/{id}/day/{n}` (CONTEXT.md D-07).
8. Map: `fitBounds()` to day's stops with 48px padding (smooth pan+zoom per CONTEXT.md D-05).
9. Non-day markers dimmed to opacity 0.35.

### Back-Navigation (breadcrumb click or browser back)

1. User clicks `Übersicht` segment in breadcrumb OR presses browser back.
2. Same crossfade sequence (150ms out → scroll top → 250ms in).
3. Breadcrumb bar hidden.
4. URL becomes `/travel/{id}`.
5. Map: `fitBounds()` to all stops (full route). All markers restored to opacity 1.0.

### Collapsible Toggle

1. User clicks `.overview-collapsible__toggle`.
2. `.overview-collapsible__body` transitions max-height 0 → 2000px (300ms).
3. Chevron rotates 180°.
4. No URL change. No scroll.

### Reduced Motion

All transitions skip (duration 0.01ms) when `prefers-reduced-motion: reduce` is set. Existing `@media (prefers-reduced-motion: reduce)` block in `styles.css` covers this automatically.

## URL Routing Contract

New routes added to `router.js`:

| Pattern | Drill Level | State |
|---------|-------------|-------|
| `/travel/{id}` | Overview (level 0) | `_activeDayNum = null`, `_activeStopId = null` |
| `/travel/{id}/day/{n}` | Day detail (level 1) | `_activeDayNum = n`, `_activeStopId = null` |
| `/travel/{id}/stop/{stopId}` | Stop detail (level 2) | `_activeDayNum` preserved, `_activeStopId = stopId` |

Browser back/forward must restore the correct drill level and re-fire map focus logic (CONTEXT.md D-07).
