### 2. API functions (frontend/js/api.js)
Add three new functions. IMPORTANT: `apiGetShared` must use plain `fetch()` (NOT `_fetchWithAuth`) because the viewer has no JWT token (per pitfall #2 in RESEARCH.md):

```javascript
async function apiGetShared(token) {
  const res = await fetch(`${API}/shared/${token}`);
  if (!res.ok) throw new Error(`HTTP ${res.status}`);
  return res.json();
}

async function apiShareTravel(travelId) {
  return _fetchQuiet(`${API}/travels/${travelId}/share`, { method: 'POST' });
}

async function apiUnshareTravel(travelId) {
  return _fetchQuiet(`${API}/travels/${travelId}/share`, { method: 'DELETE' });
}
```

### 3. Router (frontend/js/router.js)
Modify the `_travel(m)` handler. At the very top of the function, BEFORE the existing `if (S.result && ...)` check, add share token detection. When a share token is present, call `apiGetShared(shareToken)` to load the plan without auth, set `S.sharedMode = true` and `S.shareToken = shareToken`, then call `showTravelGuide(plan)`. On error, show a centered error card with heading "Link ungueltig" and body "Dieser Link ist nicht mehr gueltig oder wurde deaktiviert." using the existing `esc()` and DOM methods pattern. Then return early.

Also modify the `_travelTab`, `_travelStopDetail`, and `_travelDayDetail` handlers: at the top of each, check for `?share=` query param. If present, set `S.sharedMode = true` and `S.shareToken = shareToken`. When those handlers need to fetch the plan, branch: if `S.sharedMode`, use `apiGetShared(S.shareToken)` instead of `apiGetTravel(id)`.

**CRITICAL (per RESEARCH.md Pitfall 3 — share token URL preservation):** When `S.sharedMode` is true, ALL `Router.navigate()` calls for tab/detail routes (e.g., switching between stops/days/budget tabs, opening stop details, opening day details) MUST append `?share=${S.shareToken}` to the URL path. Without this, the share token is lost from the URL on tab clicks, breaking the shared view on page refresh or direct-link-from-tab. Implement this by either:
- (a) Wrapping or patching `Router.navigate()` to auto-append `?share=${S.shareToken}` when `S.sharedMode` is true, OR
- (b) Updating every `Router.navigate()` call site in guide.js that navigates to tab/detail routes to conditionally append the share query param.

Option (a) is preferred as it is less error-prone. If using option (a), modify the `navigate` function in router.js: before pushing state, check `if (S.sharedMode && S.shareToken)` and append `?share=${S.shareToken}` to the path if not already present.

### 5. Guide share UI (frontend/js/guide.js)
Add the share toggle rendering and management functions. Use the existing project pattern of building HTML strings with `esc()` for all user content, then setting via insertAdjacentHTML or textContent:

- `_renderShareToggle(travelId, shareToken)` returns an HTML string with:
  - A `<label class="toggle-switch">` containing `<input type="checkbox">` with `aria-label="Teilen aktivieren"` and `onchange="_handleShareToggle(travelId, this.checked)"`
  - A `<span class="toggle-slider"></span>` for the visual toggle
  - A `<span class="share-label">Teilen</span>`
  - When shared: a readonly `<input class="share-url-input">` showing the URL and a `<button class="btn btn-sm btn-secondary">Link kopieren</button>` with onclick calling `_copyShareLink`
  - All user-content values passed through `esc()`

- `_handleShareToggle(travelId, checked)` — async function:
  - If checked: call `apiShareTravel(travelId)`, re-render toggle with returned token
  - If unchecked: show `confirm('Link deaktivieren? Bestehende Empfaenger verlieren Zugriff.')` per D-10. If confirmed, call `apiUnshareTravel(travelId)` and re-render toggle with null token. If cancelled, return without changes.
  - On API error: revert toggle state and show toast "Teilen fehlgeschlagen. Bitte erneut versuchen."

- `_copyShareLink(btn, url)` — async function per D-11:
  - Call `navigator.clipboard.writeText(url)`
  - Change `btn.textContent` to `'Kopiert!'`
  - After 2 seconds (setTimeout), revert to `'Link kopieren'`
  - On clipboard error: toast "Link konnte nicht kopiert werden."

In `showTravelGuide(plan)`, after `renderGuide(plan, activeTab)`:
- If `S.sharedMode`: add `shared-mode` class to `document.body`, hide share-toggle-container
- Else: remove `shared-mode` class from `document.body`. If `plan._saved_travel_id` exists, show share-toggle-container and render the toggle with `_renderShareToggle(plan._saved_travel_id, plan.share_token || null)`
- If `S.sharedMode`: append a footer div with class `shared-footer` and textContent "Erstellt mit DetourAI" (per D-07) to the guide-content element, only if not already present
- If `S.sharedMode`: hide sidebar element (`document.querySelector('.sidebar').style.display = 'none'`)
