    **1. Add `enableClickToAdd(map, onMapClick)` to maps.js:**

    Registers a click listener on the map for empty area clicks. Marker clicks are separate events in Google Maps SDK and do not trigger this listener.

    ```javascript
    function enableClickToAdd(map, onMapClick) {
      google.maps.event.addListener(map, 'click', function(event) {
        if (event.placeId) {
          event.stop();
        }
        onMapClick(event.latLng);
      });
    }
    ```

    Export enableClickToAdd in the return object.

    **2. Add `_onMapClickToAdd(latLng)` in guide.js:**

    Called on empty map area click. Checks _editInProgress first. Hides any existing popup. Uses Google Maps Geocoder for frontend reverse geocoding (no backend round-trip per RESEARCH.md recommendation).

    ```javascript
    function _onMapClickToAdd(latLng) {
      if (_editInProgress) return;
      _hideClickToAddPopup();
      var geocoder = new google.maps.Geocoder();
      geocoder.geocode({ location: { lat: latLng.lat(), lng: latLng.lng() } }, function(results, status) {
        if (status !== 'OK' || !results || !results[0]) return;
        var placeName = results[0].formatted_address || '';
        _showClickToAddPopup(latLng, placeName);
      });
    }
    ```

    **3. Add `_showClickToAddPopup(latLng, placeName)` in guide.js:**

    Shows the #click-to-add-popup element (from index.html Plan 01) positioned at the map click point. Uses map projection to convert latLng to pixel coordinates relative to the map panel.

    The popup content uses the project's esc() function for XSS-safe escaping of the geocoded place name, following the established pattern in CLAUDE.md. Uses textContent where possible, falls back to esc() for structured HTML.

    Store the click latLng on the popup element as popup._clickLatLng for later use.
    Add Escape key listener to dismiss popup.

    **4. Add `_hideClickToAddPopup()` in guide.js:**
    Hides the popup element, clears content.

    **5. Add `_confirmClickToAdd(placeName)` in guide.js:**

    Finds best insertion point using haversine distance to determine which two consecutive stops the click falls between. Algorithm:
    - Find nearest stop by distance
    - Determine if click is before or after nearest stop by comparing distances to neighbors
    - Set insert_after_stop_id accordingly
    - Call existing _doAddStop(placeName, insertAfterStopId)

    **6. Add `_haversineKm(lat1, lon1, lat2, lon2)` utility** in guide.js:
    Standard haversine formula returning distance in km. Used only by the insert position algorithm.
