### One Partial Connection

**`_travelTab` router handler missing drill state reset** (Low severity)

The `_travelTab` handler in `router.js:268` calls `activateGuideTab(tab)` without first clearing `_activeDayNum`/`_activeStopId`. Compare with `_travel` handler (lines 205-207, 224-226) which correctly resets both. Affects NAV-04/NAV-06 only when user navigates via browser popstate to tab URLs after a prior drill — not the primary navigation path (tab buttons use `switchGuideTab` which does clear drill state).

**Recommendation:** Fix in a future polish phase. Add 2 lines to `_travelTab` handler.
